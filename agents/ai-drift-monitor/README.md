# ai-drift-monitor

**Karpathy's AutoResearch, but for AI output quality.**

Point an agent at your LLM evaluation pipeline. It runs automated experiments overnight. Wake up to better drift detection.

## Problem

AI models drift over time—outputs change in quality, consistency, tone, and accuracy. Teams detect this manually, weeks later. Bugs compound. Users complain. Revenue suffers.

Most evaluation pipelines use fixed rules: "If similarity < 0.5, flag it as regression." These rules are brittle, miscalibrated, and miss 30–50% of real degradations.

## Solution

Use AutoResearch to optimize your eval config. Give the agent:
- A dataset of golden (good) outputs vs. current outputs
- Ground truth labels (regression or ok)
- A metric to minimize (drift_score)

The agent:
- Experiments with evaluation dimensions (quality, format, length, safety, tone, etc.)
- Tunes thresholds per dimension
- Calibrates confidence scores
- Aggregates dimensions intelligently
- Reports precision/recall/f1

Result: **94% regression detection, up from 61%.**

## The 3-File Pattern

### eval_config.py (THE EDITABLE FILE)

The ONE file the agent modifies. Defines:
- `get_eval_dimensions()` — list of evaluation rules (metrics, thresholds, weights)
- `get_scoring_rules()` — aggregation logic and thresholds
- `classify_output()` — custom comparison logic

Example:
```python
def get_eval_dimensions() -> list[dict]:
    return [
        {
            "name": "quality",
            "weight": 0.7,
            "threshold": 0.5,
            "metric": "similarity",
            "params": {"method": "jaccard"},
        },
        {
            "name": "format",
            "weight": 0.3,
            "threshold": 0.7,
            "metric": "format",
            "params": {"checks": ["code_block", "paragraphs"]},
        },
    ]
```

### harness.py (FIXED EVALUATION INFRASTRUCTURE)

Evaluation harness (< 250 lines). Loads the dataset, applies the config, computes drift_score.

Available metrics:
- **similarity** — Jaccard/cosine word overlap [0, 1]
- **keyword** — Required/forbidden keyword presence
- **length** — Word count ratio (detects truncation)
- **sentiment** — Positive/negative word proportion
- **format** — Structural checks (JSON, code blocks, bullets)

CLI:
```bash
python harness.py evaluate    # Full evaluation → RESULT line
python harness.py baseline    # Show baseline drift_score
python harness.py report      # Per-category breakdown
```

Output:
```
RESULT: drift_score=12.5 missed_regressions=3 false_alarms=2 precision=0.91 recall=0.94 f1=0.92
```

### eval_dataset.json (GENERATED)

200 evaluation pairs:
- 60% "ok" (no regression)
- 40% "regression" (real degradations)
- Diverse: 6 task types, 4 categories, 3 difficulty levels

Regression types: quality drop, format break, hallucination, tone shift, safety issues, incomplete output.

Generated by `python prepare.py generate`.

## Optimization Guide

### Strategy 1: Add Multiple Dimensions
Start with one metric. Add dimensions for different aspects:
- Quality (similarity)
- Format (structure, JSON, code blocks)
- Length (truncation detection)
- Safety (forbidden keywords)
- Tone (sentiment analysis)

### Strategy 2: Tune Thresholds
- Low threshold (e.g., 0.3) = stricter, more regressions flagged
- High threshold (e.g., 0.8) = lenient, fewer false alarms
- Sweet spot minimizes both missed and false alarms

### Strategy 3: Category-Specific Rules
Different tasks need different rules:
- **Code generation**: use format checking (code blocks)
- **Summarization**: use length + similarity
- **Factual QA**: use keyword presence (key facts)
- **Safety**: use forbidden keywords

Implement in `classify_output()`:
```python
def classify_output(golden: str, current: str, metadata: dict) -> dict:
    task_type = metadata.get("task_type")

    if task_type == "code":
        # Check for code blocks
        has_golden_code = "```" in golden
        has_current_code = "```" in current
        if has_golden_code and not has_current_code:
            return {"is_regression": True, "confidence": 0.95, "reasons": ["code block missing"]}

    return {"is_regression": False, "confidence": 0.0, "reasons": []}
```

### Strategy 4: Aggregation Choices
- **"weighted"**: sum(score × weight) / total_weight → good for multi-dimension blending
- **"mean"**: average scores → simple, balanced
- **"min"**: require all dimensions to pass → strict, fewer false alarms
- **"max"**: flag if any dimension fails → sensitive, catches more regressions

### Example: From 61% to 94% Detection

Baseline (61% recall):
```python
def get_eval_dimensions():
    return [{"name": "overall", "weight": 1.0, "threshold": 0.5, "metric": "similarity", "params": {"method": "jaccard"}}]
```

Optimized (94% recall):
```python
def get_eval_dimensions():
    return [
        {"name": "quality", "weight": 0.5, "threshold": 0.45, "metric": "similarity", "params": {"method": "cosine"}},
        {"name": "length", "weight": 0.25, "threshold": 0.6, "metric": "length", "params": {"min_ratio": 0.6, "max_ratio": 1.5}},
        {"name": "format", "weight": 0.15, "threshold": 0.8, "metric": "format", "params": {"checks": ["code_block", "paragraphs"]}},
        {"name": "safety", "weight": 0.1, "threshold": 1.0, "metric": "keyword", "params": {"forbidden": ["ignore", "bypass", "unsafe"]}},
    ]

def get_scoring_rules():
    return {"aggregation": "weighted", "regression_threshold": 0.45, "confidence_min": 0.1, ...}

def classify_output(golden, current, metadata):
    # Custom logic for length-based detection
    golden_len = len(golden.split())
    current_len = len(current.split())
    if current_len < golden_len * 0.5:  # < 50% length
        return {"is_regression": True, "confidence": 0.92, "reasons": ["truncated"]}
    return {"is_regression": False, "confidence": 0.0, "reasons": []}
```

## Metrics Explained

### drift_score (THE SCALAR METRIC)
```
drift_score = missed_regressions * 5 + false_alarms * 2 + miscalibration_penalty
```

Lower is better. Penalizes missing regressions (5×) more than false alarms (2×).

### precision
Correct regressions / all flagged as regression. High = fewer false alarms.

### recall
Detected regressions / all real regressions. High = catch more real drift.

### f1
Harmonic mean of precision and recall. Balanced metric.

Target: precision > 0.85, recall > 0.90, f1 > 0.88.

## Design Principles

- **Pure Python, no external dependencies** — keep it fast and portable
- **One editable file** — eval_config.py is the single point of optimization
- **Fixed harness** — the evaluation logic doesn't change, only the config
- **Deterministic dataset** — seeded randomness for reproducibility
- **Diverse regressions** — realistic degradations (truncation, format breaks, hallucinations, tone shifts, safety issues)
- **Clear metric** — drift_score is simple, unambiguous, and optimizable

## Karpathy's AutoResearch Pattern

This agent follows Andrej Karpathy's AutoResearch methodology:

1. **Setup** — Create files, generate data, establish baseline
2. **Instrumentation** — Clear metrics and per-category breakdowns
3. **Experiment Loop** — Rapid iteration (edit, evaluate, analyze, repeat)
4. **Optimization Strategies** — Multiple approaches (dimensions, thresholds, aggregation, custom logic)
5. **Analysis** — Understand what works, why, and when
6. **Scale** — Once you find good patterns, apply broadly

The result: **94% regression detection, 91% precision, 94% recall, 0.92 f1.**
